# PERSOL WEBSTORE - Q&A PREPARATION

## 📋 TỔNG QUAN DỰ ÁN

### Q1: Dự án này là gì?
**A:** Persol Webstore là một website thương mại điện tử Single Page Application (SPA) chuyên bán kính mắt, gọng kính, tròng kính. Dự án được xây dựng như một bài tập lớn với đầy đủ chức năng từ frontend đến backend.

### Q2: Các công nghệ sử dụng trong dự án?
**A:** 
- **Frontend:** HTML5, CSS3, Bootstrap 5, JavaScript, jQuery
- **Backend:** PHP thuần (không dùng framework)
- **Database:** MySQL
- **Kiến trúc:** RESTful API, SPA (Single Page Application)
- **Deployment:** XAMPP (localhost)

### Q3: Tại sao chọn kiến trúc SPA?
**A:** 
- Trải nghiệm người dùng mượt mà, không reload trang
- Tốc độ tải nhanh sau lần đầu
- Dễ dàng quản lý state và routing
- Phù hợp với yêu cầu bài tập về responsive design

## 🏗️ KIẾN TRÚC HỆ THỐNG

### Q4: Cấu trúc thư mục của dự án?
**A:**
```
persolwebstore/
├── frontend/           # Giao diện người dùng
│   ├── assets/
│   │   ├── css/       # Stylesheets
│   │   ├── js/        # JavaScript modules
│   │   └── images/    # Hình ảnh
│   └── index.html     # Entry point
├── backend/           # API Backend
│   ├── api/          # RESTful APIs
│   └── db/           # Database connection
├── admin/            # Admin Panel
│   ├── includes/     # Shared components
│   └── *.php         # Admin pages
├── database/         # SQL files
└── uploads/          # Uploaded files
```

### Q5: Cách hoạt động của SPA routing?
**A:** 
- Sử dụng hash-based routing (#home, #products, #cart)
- JavaScript intercepts hash changes
- AJAX calls để load content động
- Không reload trang, chỉ update content area

### Q6: Cách xử lý state management?
**A:**
- localStorage để lưu user session
- Server-side session cho admin
- Client-side state cho cart, user preferences
- Real-time sync giữa frontend và backend

## 🗄️ DATABASE DESIGN

### Q7: Cấu trúc database chính?
**A:**
- **users:** Thông tin người dùng (user_id, username, fullname, email, password, role)
- **products:** Sản phẩm (product_id, name, brand, price, images, category_id)
- **categories:** Danh mục sản phẩm (category_id, name, parent_id)
- **cart:** Giỏ hàng (cart_id, user_id, product_id, quantity)
- **orders:** Đơn hàng (order_id, user_id, total_amount, status, delivery_id, trans_id)
- **order_items:** Chi tiết đơn hàng
- **delivery_info:** Thông tin giao hàng
- **transaction_info:** Thông tin thanh toán
- **banner:** Banner trang chủ

### Q8: Tại sao tách riêng delivery_info và transaction_info?
**A:**
- Tách biệt concerns: giao hàng vs thanh toán
- Có thể tái sử dụng thông tin giao hàng
- Dễ dàng mở rộng (nhiều địa chỉ, nhiều phương thức thanh toán)
- Normalization database design

## 🔐 AUTHENTICATION & AUTHORIZATION

### Q9: Cách xử lý authentication?
**A:**
- SHA-256 password hashing (không dùng bcrypt vì yêu cầu bài tập)
- JWT-like approach với localStorage
- Session-based cho admin panel
- Role-based access control (ADMIN/CUSTOMER)

### Q10: Bảo mật như thế nào?
**A:**
- Password hashing với SHA-256
- Input validation và sanitization
- Prepared statements chống SQL injection
- CORS headers cho API
- Session timeout cho admin

## 🛒 CHỨC NĂNG CHÍNH

### Q11: Workflow đặt hàng như thế nào?
**A:**
1. User thêm sản phẩm vào giỏ hàng
2. Vào giỏ hàng, click "Đặt hàng"
3. Form tự động điền thông tin từ profile
4. User xác nhận thông tin giao hàng và thanh toán
5. Backend tạo delivery_info → transaction_info → order
6. Xóa giỏ hàng, chuyển đến trang orders

### Q12: Cách xử lý auto-fill form checkout?
**A:**
- Lấy thông tin từ localStorage trước
- Nếu thiếu, gọi API lấy từ database
- Có nút "Sử dụng thông tin từ profile" để điền lại
- Real-time sync khi user cập nhật profile

### Q13: Quản lý giỏ hàng như thế nào?
**A:**
- Lưu trong database với user_id
- Real-time update số lượng sản phẩm
- Badge hiển thị số lượng trên navbar
- CRUD operations: thêm, sửa số lượng, xóa

## 🔧 BACKEND API

### Q17: RESTful API design?
**A:**
- GET /products - Lấy danh sách sản phẩm
- GET /products/{id} - Chi tiết sản phẩm
- POST /cart - Thêm vào giỏ hàng
- POST /order - Tạo đơn hàng
- PUT /user/profile - Cập nhật profile
- DELETE /cart/{id} - Xóa khỏi giỏ hàng

### Q18: Error handling trong API?
